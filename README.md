# Casual Game Template

カジュアルゲーム量産のための Unity テンプレート

## 🐟 Overview

本リポジトリは、**1画面完結型のカジュアルゲームを短期間で量産するためのテンプレート**です。
今後複数タイトルを制作する際、共通処理・UI・ゲーム進行ロジックを使い回すことで、開発速度を大幅に向上させることを目的としています。

---

## 🎯 Purpose

このテンプレートの目的は、以下の3点に集約されます。

1. **開発の高速化**
   毎タイトルごとにゼロから組み立てるコストを減らし、
   「実装：25時間 → レビュー/調整：10時間」
   のワークフローを安定して回せるようにします。

2. **品質の均一化**
   GameManager、SystemManager、UI、SE/BGM周りの処理を標準化し、
   毎回同じクオリティを担保します。

3. **量産体制の構築**
   「2週間で1アプリ」ペースを目指し、
   約 **1.16倍の効率化** を中期目標とします。

---

## 🧱 Template Structure

このテンプレートは、**1画面ゲーム**を前提に以下のシステムを内包します。

* 画面遷移を持たない1画面制御のゲームシステム
* GameManager / SystemManager
* シンプルなEventシステム（スコア更新、成長イベントなど）
* SE / BGM の再生環境
* 画像・スキン変更用の設定画面
* ステージ数は **1〜最大5** を想定（StageManagerは必要時のみ）
* 共通UI（ボタン、テキスト、ゲージ 等）

---

## 🕒 Development Plan

テンプレート完成後の開発スケジュール想定です。

* **1週間（5日）× 1日3時間 = 15時間稼働**
* **2.5週間で1ゲーム**を完成
* 目標：**2週間で1アプリ**（レビュー込み）

テンプレート初回構築は約 **5時間** を予定しています。

## 🔧 Recommended Workflow

* 新しいタイトルはこのテンプレートをクローン → 機能差分のみ実装
* Prefab と ScriptableObject を活用し、差し替えコストを最小限に
* レビューは「ゲームサイクル→UI→SE→バランス調整」の順で実施
* 全体の開発時間を35時間以内に収めることを基準とする
